React.js is a popular library for building interactive web interfaces

It was developed by facebook and released in 2013 as an open source project. Since then it has become one of the most widely used tools for front end development and was substantially expanded by community contributions and through new releases by sponsoring company.

Among other big companies using React there are: Twitter, Netflix, Airbnb, PayPal, The New York Times, Yahoo, Walmart, Uber and Microsoft.

React is designed around the concept of components, the reusable, declarable parts of web app.

Components are dynamically rendered based on their props (properties) and state.

React features functional and class components

With the same name prop passed down function and class will produce the same result Hello Alex

React projects are usually written in JSX (Javascript XML), a syntax extension of Javascript, which allows writing markup HTML elements alongside JS logic.

In a JSX file everything encapsulated in {curly brackets} behaves according to JS syntax, while ability to declare HTML elements frees React developer from the need of using DOM manipulation.

JSX code have to be compiled (usually by Babel) to pure Javascript to be processed by React's inbuilt methods into actual application.

It is theoretically possible to write React projects in pure JS calling library's methods directly, but this largely defeats the whole point of this library.

Since we mentioned DOM manipulation, another characteristic React feature is virtual DOM.

This is a process running entirely in the background through which React determines the most efficient way to update the actual DOM, thus increasing application's performance.

You might've noticed ReactDOM.render() called in the components example.

All you have to do as a developer is invoke this method on the most general component (customarily called App) and let the library handle it from there.

Advantages of using React

  • It's fairly straightforward, without a steep learning curve
  • It's flexible, can be combined with any number of other libraries or used in parts of a project only
  • It can be used in both web and mobile development
  • It comes with a large community and an ecosystem of specialized libraries and other resources
  • Virtual DOM greatly enhances performance

A couple of things to consider

  • Although not too difficult to learn it requires some getting used to
  • Projects can be tricky to set up, you may need to use additional resources to achieve desired functionality
  • A huge open-source ecosystem can be a mixed blessing. You might find yourself including libraries of various code quality and standard, sometimes with insufficient documentation

      const PlanetsList = () => {
        return (
          <div className="planets-list">
            {planets.map(planet => <Planet planet={planet} key={planet.name}/>)}
          </div>
          )
      }

Notice the curly braces surrounding pieces of javascript (both map method and then variables included inside). This code will iterate over planets array and render a <Planet /> component within <PlanetsList /> for every object in the array.

An object containing informations about given planet is passed down as prop to each <Planet /> component.

Note: in React props are only ever passed down the component hierarchy. If you need to send information back to parent component, you'll have to pass down an appropriate callback function.

Note: it is a good practice to give each element a unique {key} while using iterative methods on components. While not necessary to render content they help React inner-workings and will prevent it from throwing errors in the console.

For our example we would content ourselves with one more feature of basic interactivity. Right now the planet cards are cluttered with lots of detailed informations, so we will add an option to toggle specifics about things like mass or eccentricity with only planet name and picture displayed by default. For that we need to introduce state.

First let's add this code inside <Planet/>. A class component requires only render method called to be working, but with constructor also invoked we can define component's state. Afterwards a method is defined to change it using React's inbuilt setState()


        constructor(props){
          super(props);
          this.state = {extend: false};
        }
        
       toggle = () => {
        this.setState({
          extend: !this.state.extend
        })
      }

Now let's update Planet's render method. The js && (AND) logical operator is used for conditional rendering in React. Certain parts of our component will be displayed depending on how state is evaluated. The only thing left to do is binding toggle to click events. In the last part of this presentation we'll touch on some more advanced React concepts

<Hooks />

Starting with version 16.8 React introduces hooks, which allows to use state in functional components. All you need to do to is import {useState} from React library. Hooks address number of issues such as code reusability and readability and are generally recommended solution in today's React. That being said everything is backward compatible and class components still work throughout React versions.

In the planets example the old style class component was used but same can be achieved through hooks. You can find the alternative version commented out in the source code

<Lifecycle />

React features a host of lifecycle methods allowing to perform actions and data manipulations at specific moments during component rendering.

For example componentDidMount() runs immediately after initial render and may be used to fetch data from remote endpoint

componentDidUpdate() can be used to dynamically respond to content changes

componentWillUnmount() serves for clean-up purposes (e.g. resetting timers, cancelling web requests)

Initially available to class components, lifecycle methods now have their hook equivalents.

<Routs />

React is designed principally to build single-page application. React Router is an extension to basic library that brings along a toolkit to provide your React app with custom urls and to set up navigation between components.

<Context API />

React Context provides a way to access certain props from anywhere within components hierarchy without the need for passing them down many levels.

It is especially suited for coding things such as different styling themes or different language versions of your app.
